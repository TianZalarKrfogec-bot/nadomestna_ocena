Alternativna možnost opravljanja testa iz Flaska
Namesto klasičnega testa lahko dijak izdela 3 spletne aplikacije v Flasku.

Struktura projekta (obvezna)
Vse aplikacije morajo biti v enem Codespace projektu:

project/
│
├── app1.py
├── app2.py
├── app3.py
│
├── templates1/
├── templates2/
├── templates3/
│
├── static1/
├── static2/
├── static3/
│
└── db/
└── db2/

Vsaka aplikacija uporablja svoj templates in static folder.

Primer menjave default templates folderja
from flask import Flask, render_template

app = Flask(
    __name__,
    template_folder="templates1",
    static_folder="static1"
)

@app.route("/")
def index():
    return render_template("index.html")

if __name__ == "__main__":
    app.run(debug=True, port=5000)
Naloge
Dijak izdela naslednje tri aplikacije:

1. Osebni zapiski
več zapiskov
naslov + vsebina
pregled, urejanje, brisanje
2. Preprosta socialna platforma
objave (besedilo)
slike
pregled objav drugih uporabnikov
3. Lastna aplikacija
poljubna ideja
smiselna in dovolj kompleksna
Obvezne zahteve (za vse aplikacije)
uporaba podatkovne baze
uporaba session
uporaba sinhronih klicev (Flask route)
uporaba asinhronih klicev (AJAX)
dijak mora razumeti vso kodo

Oddaja
projekt mora biti oddan kot GitHub repozitorij (Codespace)
vidna mora biti zgodovina dela (commiti)
pričakuje se commit po vsakem večjem delu naloge
(npr. baza, login, AJAX, funkcionalnosti …)
Pomembno
rešitev mora biti samostojna
ni dovolj, da aplikacija samo deluje

delo z bazo
session

AJAX
